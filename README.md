🏟️ SportPulse — Аналітична платформа для вболівальників

SportPulse — це сучасна веб-платформа, створена для спортивних вболівальників, які хочуть приймати обґрунтовані рішення, глибоко аналізувати матчі та спілкуватися з однодумцями. Ми поєднуємо машинне навчання, детальну статистику та активну спільноту в одному зручному та швидкому інтерфейсі.

---

 🚀 Головні функції

1. 🤖 ШІ-прогнози на матчі — Алгоритми на основі історичних даних, поточної форми команд, погодних умов та інших факторів генерують прогнози з прозорою оцінкою ймовірності.
2. 📊 Аналітична панель у реальному часі — Відстежуйте ключові показники, хвилинну статистику, травми, стартові склади та динаміку коефіцієнтів без перезавантаження сторінки.
3. 👥 Вбудоване ком'юніті — Тематичні форуми, живі чати під час матчів, рейтинги прогнозів, опитування та можливість публікувати власні аналітичні розбори.
4. 🔔 Персональні сповіщення та трекер улюблених клубів — Гнучке налаштування подій, push/email-нагадування перед стартовим свистком, зміною складців або аномальними рухами коефіцієнтів.
5. 🏆 Система рейтингу та експертні огляди — Змагайтеся з іншими вболівальниками за точність, отримуйте бейджі, відкривайте преміум-інсайти та читайте щоденні підбірки від професійних аналітиків.

 🏁 Як запустити

Вимоги
- Node.js `>=18`
- Python `>=3.11`
- Docker & Docker Compose (рекомендовано)

🐳 Швидкий старт через Docker
```bash
git clone https://github.com/ВАШ_НІК/sportpulse.git
cd sportpulse
cp .env.example .env
docker-compose up -d --build
```
Після запуску платформа буде доступна за адресою: `http://localhost:3000`

💻 Ручний запуск (для розробки)
1. Backend:
   ```bash
   cd backend
   python -m venv venv
   source venv/bin/activate  Windows: venv\Scripts\activate
   pip install -r requirements.txt
   uvicorn main:app --reload --host 0.0.0.0 --port 8000
   ```
2. Frontend:
   ```bash
   cd frontend
   npm install
   npm run dev
   ```
3. Відкрийте `http://localhost:3000` у браузері.

> 💡 Обов'язково налаштуйте змінні оточення у `.env` (ключі зовнішніх API, параметри підключення до БД, `JWT_SECRET`, тощо). Приклади доступні у `.env.example`.
